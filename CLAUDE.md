# Portfolio — Claude Instructions

## Branch workflow

- Default development branch is `dev`. Always work on `dev` unless explicitly told otherwise.
- Never push to `main` without explicit instruction in the current session.
- When starting work, ensure `dev` is checked out and up to date (`git pull origin dev`).

## Portfolio content — primary audience

The target audience for all portfolio content is **recruiters and product design managers**. Apply this to every content change:

- **Copy**: succinct and demonstrates design thinking — decisions and rationale, not just a list of deliverables. Show what was hard, what changed, and why.
- **Structure**: follow the established format of existing project pages (problem → callout → process → my role → solution → impact). Don't add extra sections unless explicitly asked.
- **Images**: use clean UI mockups or screens only. Never use presentation slides, annotated process artifacts, or images with slide chrome (headings, labels, layout frames) unless explicitly requested.
